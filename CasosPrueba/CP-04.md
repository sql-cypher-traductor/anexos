| **ID**                       | CP-04                               |
| **Historia de Usuario**      | QTE-03                              |
| **Nombre**                   | Traducir consultas con `INNER JOIN` |
| **Cumple (Sí/No)**           | Sí                                  |
| **Descripción de la Prueba** | Validar la traducción correcta de `INNER JOIN` a una relación entre nodos. |
| **Precondiciones**           | 1. El usuario debe estar autenticado en el sistema<br>2. El usuario debe tener una conexión a Neo4j almacenada y activa. |
| **Pasos**                    | 1. Seleccionar una conexión a Neo4j activa<br>2. Ingresar la sentencia válida<br>3. Dar clic en el botón `Traducir` |
| **Datos de Prueba**          | Sentencia: `SELECT o.OrderID, c.CompanyName, o.OrderDate FROM Orders o INNER JOIN Customers c ON o.CustomerID = c.CustomerID WHERE o.OrderDate >= '1997-01-01' ORDER BY o.OrderDate DESC;` |
| **Resultados Esperados**     | El sistema indica el equivalente en lenguaje Cypher junto a la sentencias SQL ingresada. |
| **Resultados Obtenidos**     | El sistema indica un mensaje de traducción exitosa y su equivalente en lenguaje Cypher. |
